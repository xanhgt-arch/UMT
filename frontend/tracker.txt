19. FIX "THIS YEAR" MONTHLY CHART (WRONG MONTHS)
----------------------------------------------------------------
Request (with screenshot): When "This year" is selected and the
current month is May 2026, the Monthly Usage chart shows Aug-Dec
instead of Jan-May.

Root cause was two stacked bugs:
  (a) MONTHLY_USAGE / MONTHLY_CAD_USAGE were being built as a
      fixed Jan-Dec calendar array. rangeSlice() in filtering.ts
      was tail-based (slice(-(currentMonth+1))), which assumes
      a rolling 12-month window ending at the current month.
      Slicing the last 5 of a Jan-Dec array gives Aug-Dec, not
      Jan-May.
  (b) Even with (a) fixed, the calendar shape conflated 2025
      and 2026 sessions into the same buckets, so "Jan"
      contained Jan 2025 + Jan 2026 piled together. Selecting
      "This year" would still mix years.

  - src/lib/mock-data.ts
      Replaced the Jan-Dec arrays with a true rolling 12-month
      window. ROLLING_BUCKETS is built from today's calendar
      month going back 11 months; each bucket carries (year,
      month, label). Labels include the two-digit year
      ("Jan '26", "May '26") so the year boundary is visible
      on the X-axis when the window straddles years.
      Session aggregation now keys on (year, month) via a
      bucketIndex Map, so a 2025 January session lands in a
      different bucket than a 2026 January session.
  - src/lib/filtering.ts
      rangeSlice() doc comment tightened to describe the
      rolling shape ("oldest -> newest, newest = current
      month"). The slice math itself didn't need to change -
      it was already correct for a rolling window.
  - scripts/__smoke-monthly.mjs (new, dev-only)
      Prints the full 12-bucket rolling window and the slices
      that "This year / This month / Last month" would return.
      Verified: This year = Jan '26..May '26; This month =
      May '26; Last month = Apr '26.

28. SMART SEGMENT LABELS — CALLOUTS FOR TINY VALUES
----------------------------------------------------------------
Bug (screenshot): In the region stack, Asia's NX segment was 16
sessions next to 3,340 CATIA. The "16" label was rendered inside
the segment and clipped because the slice was too narrow to fit
the text. Same problem on the monthly region chart (May's tiny
slice).

Approach: When a segment is big enough for its value, render the
label inside as usual; otherwise pop it out of the segment as a
callout in the segment's own colour with a short leader line
pointing back to it. Two leader geometries — one for vertical
bar stacks (callout to the right), one for horizontal bar stacks
(callout above with an L-shaped tick).

New shared module:
  - src/components/dashboard/charts/segment-label.tsx
      Exports two helpers:
        segmentLabelVertical({color, insideFill?})   — vertical
          stacks. Inside fits when segment height >= 14 px.
          Callout: horizontal leader to the right of the bar.
        segmentLabelHorizontal({color, insideFill?}) — horizontal
          stacks. Inside fits when segment width >= text width
          (~6.6 px/char + 10 px padding at 12 px font).
          Callout: L-shaped leader above the bar.
      `color` is used for the callout text + leader; `insideFill`
      defaults to white but can be overridden (used for light
      brand segments like chart-2 / chart-4 where white reads
      poorly against the fill).

Applied to:
  - src/components/dashboard/charts/RegionBars.tsx
      Inline segmentLabel() helper removed; CATIA / NX / Other
      LabelLists now consume segmentLabelHorizontal(color).
      Chart top margin bumped 8 -> 24 to give callouts headroom.
  - src/components/dashboard/charts/RegionMonthly.tsx
      Inline stackedSegmentLabel() helper removed; stacked-mode
      LabelLists now consume segmentLabelVertical(REGION_COLOR[r]).
      Per-region inline values also added to grouped + line
      modes (LabelList position="top" in the region's own colour
      with zero-value suppression).
  - src/components/dashboard/charts/MonthlyUsageTotal.tsx
      Old renderStackLabel (which silently hid too-small
      segments) replaced with segmentLabelVertical for CATIA
      and NX. insideFill swapped per segment so light NX uses
      foreground text while dark CATIA stays white.
  - src/components/dashboard/charts/ApplicationBars.tsx
      Same swap for all four segments (Validation, Execution,
      Block creation, Viewing). Light segments (chart-2,
      chart-4) carry insideFill="var(--foreground)".
